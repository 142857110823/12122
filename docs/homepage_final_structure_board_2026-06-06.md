# 首页最终设计结构板

## 1. 目标定位
- 首页是平台门面页，不是操作页，不是结果页，也不是流程说明页。
- 首页只承担两类信息：平台定位、水蚀科研认知。
- 首页要体现专业性、学术气质和视觉完成度，但不能把下游模块内容提前堆回首页。

## 2. 首页边界
### 必须保留
- 平台名称
- 一句核心定位
- 2 到 3 个研究标签
- 黑土区水蚀研究的核心认知
- 一个可折叠的补充阅读区

### 必须删除或禁止出现
- 数据处理说明
- 模型训练说明
- 结果解释说明
- 预测导出说明
- 工作流、门禁、阶段说明
- 模型指标、结果摘要、运行状态
- AI 聊天主入口说明文案
- “点击开始”“立即训练”这类产品化营销话术

## 3. 最终信息架构
### 第一层：主视觉区
- 左侧：
  - 平台标签
  - 平台主标题
  - 一句副标题
  - 研究标签 3 个
- 右侧：
  - 学术型抽象视觉
  - 不承载数据或结果，只承载专业气质

### 第二层：科研认知区
- 三张并列认知卡，分别固定为：
  - 研究对象
  - 五因子体系
  - 空间表达
- 每张卡只允许：
  - 1 个标题
  - 1 段短说明
  - 1 条强调句

### 第三层：补充阅读区
- 统一放入 1 个 `expander`
- 只放背景性、术语性、适用范围内容
- 默认收起

## 3.2 组件合同
### Hero Block
- 必须包含：标签、标题、副标题、标签组、1句边界说明。
- 不得包含：操作按钮、运行状态、模型数字、工作流解释。
- 首页不得在 Hero 之前再出现单独的“页面说明卡”或重复标题壳，避免首屏从门面页退回后台工作页。

### Academic Visual
- 必须是学术型抽象视觉，不是营销插画。
- 可以使用：地貌纹理、流域线感、小型信息面板、因子框架字样。
- 不得使用：模型分数、地图预览、导出入口。

### Focus Cards
- 数量固定为 3 张。
- 标题固定围绕：
  - 研究对象
  - 五因子体系
  - 空间表达
- 每张卡文本上限：
  - 1段主体说明
  - 1条强调句

### Reading Expander
- 只能承担补充阅读角色。
- 默认收起。
- 不扩展成第二层功能说明区。

## 3.3 首页模块配置蓝图
### 上层固定壳
- 顶部导航固定保留：`首页 / 数据处理 / 模型构建 / 结果解释 / 预测导出`
- 左侧边栏固定保留：上传器、当前文件状态、重置入口
- 本地助手固定保留：悬浮于上层，但不侵入首页主体信息流

### 首页主体只允许 3 个模块
1. `Hero 模块`
- 宽度比例建议为 `文本 58% - 62% / 视觉 38% - 42%`
- 文本区只放：平台标签、标题、副标题、研究标签、边界说明
- 视觉区只放：学术型抽象视觉与 4 个小型认知面板

2. `科研认知模块`
- 只允许 3 张卡并列
- 卡片顺序固定：`研究对象 -> 五因子体系 -> 空间表达`
- 卡片职责固定为“建立理解框架”，不是“说明业务操作”

3. `补充阅读模块`
- 固定为单一 `expander`
- 仅承接背景、术语、适用范围、方法学边界
- 默认收起，避免破坏首屏节奏

### 模块间距与密度原则
- Hero 与认知卡之间必须有明显层级断点
- 单张卡正文建议控制在 `70 - 110` 汉字
- 首页默认可见正文总量建议控制在 `260 - 360` 汉字
- 任何新增内容都必须回答：它是在强化平台身份，还是在偷渡下游功能说明

## 3.4 丰富性来源
### 应该增加的“丰富性”
- 更强的标题层级
- 更稳的留白比例
- 更学术化的抽象视觉
- 更清晰的认知分块
- 更克制但更准确的专业术语

### 不应该增加的“丰富性”
- 更多功能入口
- 更多结果卡片
- 更多状态提示
- 更多工作流解释
- 更长的段落说明
- 任何首页可见的模型指标、图件、排行榜、门禁表

## 4. 视觉方向
- 主色：深绿 + 土壤暖棕 + 暖灰白
- 风格：科研平台、地貌感、低饱和、留白充足
- 首页丰富性来自：
  - 主视觉构图
  - 卡片层级
  - 学术型装饰图形
- 不依赖：
  - 大量文字堆砌
  - 大量功能卡片
  - 花哨插画

## 5. 文案方向
### 主标题
- 黑土区水蚀智能预测与证据化评估平台

### 副标题
- 面向黑土区水蚀过程认知、因子解释与空间表达的科研分析平台。

### 标签
- 黑土区
- RUSLE五因子
- 空间解释

### 三张认知卡
- 研究对象：说明黑土区水蚀关注的地形、降雨、覆盖与管理作用。
- 五因子体系：说明 R、K、LS、C、P 作为认知框架，而不是结果展示。
- 空间表达：说明平台为何强调主导因子识别与空间异质性表达。

## 6. 执行方向
### 本轮执行重点
- 先收紧首页，不改动数据处理模块。
- 首页从“双说明卡”升级为“主视觉 + 三卡 + 折叠阅读”。
- 保持顶部导航、侧边栏、本地助手不变。
- 首页文案与视觉语义统一为中文科研表达，不保留宣传式或半英文半中文的杂糅感。

### 本轮不做
- 不把结果区搬回首页
- 不在首页增加操作按钮
- 不在首页展示真实模型或预测产物
- 不把首页改成营销落地页

### 后续迭代方向
- 先提升首页主视觉的学术表达力，再考虑局部动效。
- 如果继续增强首页，只允许增强：
  - 主视觉质感
  - 三卡排版
  - 字体层级
  - 小型抽象信息图
- 不允许通过增加文字块或功能块来伪装“更丰富”。
- 如果后续需要新增首页元素，必须先写入此结构板，再进入代码实现。

## 7. 验收标准
### A. 信息边界验收
- 首页默认可见区域不得出现数据处理、模型训练、结果解释、预测导出的操作说明。
- 首页不得出现工作流、门禁、模型结果、运行证据等执行层信息。

### B. 结构验收
- 首页必须清晰呈现三层结构：
  - 主视觉区
  - 三张认知卡
  - 一个补充阅读折叠区
- 首屏进入页面后，`Hero` 必须是默认注意力中心，不允许被重复页头、说明卡或状态条抢占。

### C. 视觉验收
- 首屏必须具备明确标题层级和视觉锚点。
- 页面不能只剩纯说明文字，也不能堆满卡片导致拥挤。
- 主视觉与认知卡之间要有明显层级差。

### C1. UI 结构验收
- 首屏必须首先看到主视觉区，而不是说明卡先占据注意力。
- 三张认知卡必须在同一视觉层，且不混入操作入口。
- 折叠阅读区必须位于认知卡之后，不能上移到主视觉之前。
- 顶部导航可以存在，但只能作为全局壳；不能再叠加一个与 Hero 重复表达的平台标题容器。

### D. 交互验收
- 顶部导航继续可用。
- 侧边栏继续常驻。
- 本地助手继续悬浮，不干扰首页主视觉阅读。

### E. 代码与运行验收
- `app.py` 编译通过。
- UI 单测通过。
- 浏览器预览可正常打开首页且布局稳定。

### E1. 证据清单
- 结构板文件存在并可供后续模块复用。
- 首页代码实现已落在 `render_home()`。
- 最新首页截图应作为预览证据保存到 `data/runs/.../home.png`。

## 8. 偏航判定
出现以下任一情况，视为方向走偏：
- 首页重新出现流程说明或功能堆叠
- 首页出现模型、指标、结果摘要
- 首页依靠大量文字假装“丰富”
- 首页视觉像普通后台说明页而非科研平台门面页
- 首页首屏重新出现“标题壳 + 页面说明卡 + Hero”三层重复表达
